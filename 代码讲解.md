# 第一部分：编程前置知识

在看代码之前，我们先把用到的编程概念全部讲一遍。每个概念会用生活中的例子来类比。

---

## 1.1 什么是编程

**编程**就是给计算机写一份"指令清单"，告诉它一步一步该做什么。就像你写一份菜谱：

```
第一步：打开冰箱，拿出鸡蛋
第二步：把鸡蛋打到碗里
第三步：搅拌均匀
第四步：开火，倒油
第五步：把蛋液倒入锅中
第六步：等30秒，翻面
第七步：关火，装盘
```

计算机就是那个"厨师"，它会严格按照你写的步骤执行。菜谱写错了（比如忘了说开火），做出来的菜就会出问题——程序也是一样。

**Java** 是一种编程语言，就像中文、英文一样，是人和计算机沟通的"语言"。我们用 Java 写的代码，最终会被翻译成计算机能理解的指令。

---

## 1.2 什么是变量

**变量**就是一个"盒子"，用来存放数据。你可以给盒子贴一个标签（变量名），然后把东西放进去。

```
生活中的例子：
你有一个盒子，标签写着"今天的气温"，里面放了一张纸条写着"35度"
```

在 Java 里，写法是：

```java
int temperature = 35;
// int    = 盒子的类型（放整数的盒子）
// temperature = 盒子的标签（变量名）
// 35     = 盒子里放的东西（变量的值）
// ;      = 句号（Java 每句话都要以分号结尾）
```

常用的盒子类型：
| 类型 | 能放什么 | 例子 |
|------|----------|------|
| `int` | 整数 | `int age = 20;` |
| `String` | 一串文字（字符串） | `String name = "张三";` |
| `double` | 小数 | `double price = 9.99;` |
| `Integer` | 整数（对象版） | `Integer id = 1;` |

`int` 和 `Integer` 的区别：`int` 是最基础的数字类型，`Integer` 是它的"升级版"，可以在特殊情况下表示"没有值"（null）。项目里用 `Integer` 是因为数据从文件读出来时可能是空的。

---

## 1.3 什么是数据类型

数据类型就是告诉计算机"这个盒子里要放什么类型的东西"。就像你不能把液体倒进一个方盒子里——容器的形状决定了它能装什么。

Java 有两大类数据类型：

- **基本类型**：`int`、`double`、`char`（单个字符）、`boolean`（真/假）——这些是最简单的值。
- **引用类型**：`String`（文字）、`List`（列表）、你自定义的类（如 `User`）——这些是复杂的数据结构。

---

## 1.4 什么是类和对象

**类（Class）** 是一张"设计图纸"，**对象（Object）** 是根据图纸造出来的"实物"。

```
生活中的例子：
"汽车"是一个类（设计图纸），上面写了汽车有什么特征（品牌、颜色、速度）和能做什么（启动、加速、刹车）。
"我那辆红色的宝马"是一个对象（实物），它有具体的值（品牌=宝马，颜色=红色，速度=0）。
```

在 Java 里：

```java
// 这是"图纸"（类）
public class User {
    String username;   // 特征：用户名
    String password;   // 特征：密码
}

// 这是"实物"（对象）
User u = new User();          // 创建一个空的用户
u.username = "张三";           // 给它的用户名赋值
u.password = "123456";        // 给它的密码赋值
```

- `class` 关键字表示"我要定义一个类"。
- `User` 是类名（图纸的名字）。
- `new User()` 是创建一个对象（按图纸造一个实物）。
- `u.username = "张三"` 是给对象的属性赋值。

---

## 1.5 什么是方法

**方法（Method）** 就是"一段可以反复使用的指令"，类似于菜谱中的一个独立步骤模块。

```
生活中的例子：
"炒鸡蛋"是一个方法：
  输入：鸡蛋、油、盐
  步骤：打蛋→搅拌→热油→倒入→翻炒→出锅
  输出：一盘炒鸡蛋

你可以在不同的菜里调用"炒鸡蛋"这个方法，不用每次都重新写步骤。
```

在 Java 里：

```java
// 定义一个方法
public String greet(String name) {
    return "你好，" + name;
}

// 调用这个方法
String result = greet("张三");
// result 的值变成 "你好，张三"
```

方法的组成部分：
- `public` = 这个方法是公开的，其他地方可以调用它。
- `String` = 这个方法执行完后会返回一个文字结果。
- `greet` = 方法的名字。
- `(String name)` = 输入参数，调用时需要传入的值。
- `return` = 把结果送回去。

---

## 1.6 什么是 import

`import` 就是"借别人写好的工具"。Java 自带了很多工具库，你需要用 `import` 告诉计算机"我要用这个工具"。

```java
import java.io.BufferedReader;    // 借一个"读文件的工具"
import java.util.ArrayList;       // 借一个"列表工具"
import javax.swing.*;             // 借所有"图形界面工具"（*表示全部）
```

就像做菜时你不会从头造一把菜刀，而是去工具箱里拿一把现成的。

## 1.7 什么是继承

**继承**就是一个类"继承"另一个类的全部特征，然后再加上自己的新特征。

```
生活中的例子：
"动物"有特征：名字、年龄
  ↓ 继承
"狗"有特征：名字、年龄（从动物继承来的）+ 品种（自己新加的）
  ↓ 继承
"导盲犬"有特征：名字、年龄、品种（从狗继承来的）+ 工作证书号（自己新加的）
```

在 Java 里：

```java
public class LoginFrame extends JFrame {
    // LoginFrame 继承了 JFrame 的所有功能
    // JFrame 提供了"窗口"的基本能力（标题栏、关闭按钮、大小设置）
    // LoginFrame 在此基础上加了自己的功能（登录按钮、注册按钮）
}
```

- `extends` 关键字表示"继承自"。
- `JFrame` 是 Java 自带的"窗口类"，提供了创建窗口需要的所有基础功能。
- `LoginFrame` 继承 `JFrame` 后，自动拥有了窗口的能力，只需要关注登录相关的逻辑。

## 1.8 什么是封装

**封装**就是把数据"锁起来"，只留几个"窗口"（方法）来读取和修改。

```
生活中的例子：
银行账户：
  你的余额被锁在银行保险柜里（private，外部不能直接碰）
  你想查余额 → 走"查询窗口"（getter 方法）
  你想存钱   → 走"存款窗口"（setter 方法）
  不能直接去保险柜里改数字
```

在 Java 里：

```java
public class User {
    private String username;  // private = 被锁起来了，外部不能直接访问

    // "查询窗口"：允许外部读取 username 的值
    public String getUsername() {
        return username;
    }

    // "存款窗口"：允许外部修改 username 的值
    public void setUsername(String username) {
        this.username = username;
    }
}
```

- `private` = 私有的，只有这个类内部能访问。
- `public` = 公开的，外部可以调用。
- `getUsername()` = getter 方法，读取值。
- `setUsername(...)` = setter 方法，修改值。
- `this.username` = `this` 指的是"当前这个对象"，`this.username` 就是"当前对象的 username"。

## 1.9 什么是循环

**循环**就是重复做同一件事，直到条件不满足为止。

```
生活中的例子：
"把书架上每本书的书名写到纸上"：
  拿第一本书 → 写书名 → 放回去
  拿第二本书 → 写书名 → 放回去
  拿第三本书 → 写书名 → 放回去
  ...直到没有书了，停止
```

Java 里最常用的循环：

```java
// for 循环：知道要重复几次
for (int i = 0; i < 5; i++) {
    System.out.println("第" + i + "次");
}
// 输出：第0次、第1次、第2次、第3次、第4次

// while 循环：不知道要重复几次，条件满足就一直做
while (还有书) {
    拿一本书 → 写书名 → 放回去
}
```

## 1.10 什么是条件判断

**条件判断**就是"如果……就……，否则……"。

```
生活中的例子：
如果下雨 → 带伞
否则     → 不带伞
```

在 Java 里：

```java
if (下雨) {
    带伞();
} else {
    不带伞();
}

// 也可以用 else if 判断多个条件
if (成绩 >= 90) {
    评级 = "优秀";
} else if (成绩 >= 60) {
    评级 = "及格";
} else {
    评级 = "不及格";
}
```

`==` 表示"等于"，`!=` 表示"不等于"，`>` 表示"大于"，`<` 表示"小于"，`>=` 表示"大于等于"，`&&` 表示"并且"（两个条件都要满足）。

## 1.11 什么是异常处理

程序运行时可能会出错（比如文件不存在、用户输入了文字但代码期望数字）。**异常处理**就是"提前准备好出错时的应对方案"，防止程序直接崩溃。

```
生活中的例子：
你去餐厅点菜：
  正常流程：点菜 → 做菜 → 上菜
  异常情况：点的菜没有 → 告诉你没有这道菜，推荐其他的
  如果没有处理异常：厨房直接起火了（程序崩溃）
```

在 Java 里：

```java
try {
    // 尝试做可能出错的事
    int num = Integer.parseInt("abc");  // 把文字转成数字，"abc"不是数字，会出错
} catch (Exception e) {
    // 出错了怎么办
    System.out.println("输入不是数字！");
}
```

- `try` = "试试看"
- `catch` = "如果出错了"
- `e.printStackTrace()` = 把错误信息打印出来（方便调试）

## 1.12 什么是集合（List、Map）

**集合**就是"可以装很多东西的容器"。

### List（列表）

**List** 就像一个"有序的收纳盒"，东西按顺序排列，可以一个一个拿出来。

```
生活中的例子：
购物清单：
  第1个：牛奶
  第2个：面包
  第3个：鸡蛋
  可以按顺序一个一个看，也可以加新的进去
```

```java
List<String> list = new ArrayList<>();  // 创建一个空列表
list.add("牛奶");                       // 加入第一个
list.add("面包");                       // 加入第二个
list.add("鸡蛋");                       // 加入第三个

String first = list.get(0);  // 取第1个（Java 从0开始数：0、1、2）
// first = "牛奶"

int size = list.size();      // 列表里有几个元素
// size = 3
```

### Map（映射）

**Map** 就像一本"字典"，通过一个词（key）快速找到它的解释（value）。

```
生活中的例子：
字典：
  "苹果" → 一种水果
  "狗"   → 一种动物
  通过"苹果"这个词快速找到"一种水果"
```

```java
Map<Integer, String> map = new HashMap<>();
map.put(1, "默认");   // 放入：编号1 → "默认"
map.put(2, "同学");   // 放入：编号2 → "同学"

String name = map.get(1);  // 通过编号1查找
// name = "默认"
```

## 1.13 什么是 Lambda 表达式

Lambda 表达式就是"一种简写方式"，把一段简短的代码写得更紧凑。

```java
// 传统写法（比较长）
button.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        System.out.println("按钮被点击了");
    }
});

// Lambda 写法（简短）
button.addActionListener(e -> {
    System.out.println("按钮被点击了");
});
```

- `e` 是事件对象（按钮被点击时产生的信息）。
- `->` 表示"执行后面的操作"。
- `{ }` 里面是要执行的代码。

本项目中所有按钮点击事件都用 Lambda 写法。

## 1.14 什么是 Stream API

Stream API 是 Java 8 新增的功能，用来对"一批数据"做批量处理，代码更简洁。

```
生活中的例子：
你有一篮子水果，想找出所有红色的：
  传统方式：一个一个拿出来看，红色的放左边，不是红色的放右边
  Stream 方式：把水果倒进一个流水线，流水线自动把红色的筛选出来
```

```java
// 假设有一个名字列表，要找出所有以"张"开头的
List<String> names = List.of("张三", "李四", "张伟", "王五");

List<String> result = names.stream()       // 把列表变成流水线
    .filter(name -> name.startsWith("张")) // 过滤：只留以"张"开头的
    .collect(Collectors.toList());          // 收集回列表

// result = ["张三", "张伟"]
```

- `.stream()` = 启动流水线。
- `.filter(...)` = 过滤，条件满足的留下。
- `.collect(...)` = 收集结果。
- 本项目在查询和删除操作中大量使用。

## 1.15 什么是图形界面（GUI）

**GUI（Graphical User Interface，图形用户界面）** 就是你在屏幕上看到的窗口、按钮、输入框这些东西。

```
传统程序（命令行）：
  请输入用户名：___  （你在黑色窗口里打字）
  请输入密码：___
  按回车登录

图形界面程序：
  ┌──────────────────┐
  │  账号：[输入框]    │
  │  密码：[输入框]    │
  │  [登录] [注册]     │  ← 用鼠标点击按钮
  └──────────────────┘
```

Java 自带一个叫 **Swing** 的图形界面工具箱，提供了各种现成的组件：

| 组件 | 作用 | 本项目怎么用 |
|------|------|------------|
| `JFrame` | 窗口（整个窗口本身） | LoginFrame、MainFrame 都继承它 |
| `JButton` | 按钮 | "登录"、"新增"等按钮 |
| `JTextField` | 单行输入框 | 输入用户名、姓名等 |
| `JPasswordField` | 密码输入框（输入内容显示为*） | 输入密码 |
| `JLabel` | 标签（显示文字） | "账号："、"密码："等提示文字 |
| `JTable` | 表格 | 显示联系人列表 |
| `JScrollPane` | 滚动条 | 表格内容多时可以滚动 |
| `JComboBox` | 下拉选择框 | 选择分组 |
| `JOptionPane` | 弹出对话框 | 提示信息、确认删除等 |
| `GridLayout` | 网格布局（把组件排成几行几列） | 登录界面的布局 |
| `BorderLayout` | 边界布局（上/下/左/右/中） | 主界面的布局 |

## 2.4 数据是怎么存的

本项目用 **txt 文本文件** 存储数据，不用数据库。每行一条记录，字段之间用**逗号**分隔。

### user.txt — 用户表

```
1,123,qqqqqqqq
2,123456,123456
3,2323,2323
```

每行格式：`ID,用户名,密码`

- 第1列：用户的唯一编号（自动生成）
- 第2列：登录用户名
- 第3列：登录密码

### group.txt — 分组表

```
1,默认,1
2,同学,0
3,12,0
```

每行格式：`ID,分组名称,是否默认`

- 第1列：分组的唯一编号
- 第2列：分组名称
- 第3列：`1` 表示是默认分组（不可删除），`0` 表示普通分组

### contact.txt — 联系人表

```
1,张三,11111111111,浙江,丽水,莲都,丽水学院,zhagsan@qq.com,null,3,1
2,李四,22222222222,江西,上饶,玉山,无,无,无,1,1
```

每行格式：`ID,姓名,电话,省份,城市,区县,单位,邮箱,备注,分组ID,用户ID`

- 第1列：联系人的唯一编号
- 第2-9列：联系人的各种信息
- 第10列：所属分组的编号（关联 group.txt）
- 第11列：所属用户的编号（关联 user.txt，实现用户隔离）

### 数据之间的关系

```
user.txt 的 ID ←── contact.txt 的第11列（用户ID）
                     每个联系人属于一个用户

group.txt 的 ID ←── contact.txt 的第10列（分组ID）
                     每个联系人属于一个分组
```

```java
    }

    // 读取 username 的值
    // 调用方式：u.getUsername()  → 返回用户名
    public String getUsername() {
        return username;
    }

    public String getPassword() {
        return password;
    }


    // ========== Setter 方法（修改属性的值）==========

    // 调用方式：u.setId(5)  → 把 id 改成 5
    public void setId(Integer id) {

    }

    }

        this.password = password;
    }

```

### 为什么要用 private + getter/setter？

这是 Java 的**封装**原则。如果字段是 public 的，任何人都可以直接改：
```java
u.password = "hacked";  // 危险！任何代码都能改密码
```
用 private 后，只能通过 setter 方法改，你可以在 setter 里加校验逻辑（比如密码长度检查），虽然本项目没加，但这是好的编程习惯。

## 3.5 FileUtil.java — 文件读写工具

**文件路径**：`src/FileUtil.java`
**总行数**：56 行
**作用**：提供所有文件读写操作的方法，其他类不直接操作文件，而是通过这个类间接操作。相当于一个"文件操作的工具箱"。

### 完整代码 + 逐行注释

```java
// import 引入需要用到的工具
import java.io.*;              // Java 的文件操作工具包（包含 File、BufferedReader、FileReader 等）
import java.util.ArrayList;    // ArrayList 是 List 的一种实现，可以动态增减容量的列表
import java.util.List;         // List 是列表的"接口"（规范），规定了列表应该有哪些方法


// 定义一个公开的类，类名叫 FileUtil
public class FileUtil {

    // ========== 文件路径常量 ==========

    // "public static final" 表示这是一个公开的、静态的、不可修改的常量
    // 常量名用全大写 + 下划线，这是 Java 的命名惯例
    // "USER_FILE" 的值是 "user.txt"，表示用户数据文件的路径
    // 路径是相对路径，相对于项目根目录（即 JavaHelloWorld 文件夹）
    public static final String USER_FILE = "user.txt";
    public static final String GROUP_FILE = "group.txt";
    public static final String CONTACT_FILE = "contact.txt";


    // ========== 静态初始化块 ==========

    // "static { }" 是 Java 的静态初始化块
    // 它在类第一次被使用时自动执行，而且只执行一次
    // 作用：确保三个 txt 文件在程序启动时就存在
    static {
        createFile(USER_FILE);       // 确保 user.txt 存在
        createFile(GROUP_FILE);      // 确保 group.txt 存在
        createFile(CONTACT_FILE);    // 确保 contact.txt 存在

        // 如果分组文件是空的，就写入一个默认分组
        if (readAllLines(GROUP_FILE).isEmpty()) {
            writeLine(GROUP_FILE, "1,默认,1");
        }
    }


    // ========== createFile 方法：创建文件（如果不存在的话）==========

    // 参数：String path = 文件路径（如 "user.txt"）
    private static void createFile(String path) {
        // new File(path) = 创建一个"文件对象"，代表这个路径对应的文件
        File f = new File(path);
        try {
            // f.exists() = 检查这个文件是否存在
            // 如果不存在，就创建它
            if (!f.exists()) f.createNewFile();
        } catch (IOException e) {
            // IOException 是"输入输出异常"，文件操作可能出的错
            // e.printStackTrace() = 把错误信息打印到控制台（方便调试）
            e.printStackTrace();
        }
    }


    // ========== readAllLines 方法：读取文件的所有行 ==========

    // 参数：String path = 文件路径
    // 返回值：List<String> = 一个列表，里面装着文件的每一行内容
    public static List<String> readAllLines(String path) {
        // 创建一个空的列表，用来装读到的内容
        List<String> list = new ArrayList<>();

        // try(BufferedReader br = ...) = 创建一个"读文件的工具"，用完自动关闭
        // BufferedReader = 缓冲读取器，可以一行一行地读文件
        // FileReader = 文件读取器，负责打开文件
        try (BufferedReader br = new BufferedReader(new FileReader(path))) {
            String line;  // 用来临时存放每一行的内容

            // br.readLine() = 读一行内容
            // 如果读到文件末尾（没有更多内容了），返回 null
            // while 循环：只要读到的内容不是 null，就继续读
            while ((line = br.readLine()) != null) {
                // .trim() = 去掉字符串首尾的空格和换行符
                // 例如 "  hello  " → "hello"
                String trimLine = line.trim();

                // 如果这一行不是空的（去掉空格后还有内容），才加入列表
                if (!trimLine.isEmpty()) {
                    list.add(trimLine);
                }
            }
        } catch (IOException e) {
            e.printStackTrace();  // 读取出错时打印错误信息
        }

        // 返回读到的所有行
        return list;
    }


    // ========== writeAll 方法：覆盖写入文件 ==========

    // 参数：String path = 文件路径，List<String> data = 要写入的内容（多行）
    // 效果：清空文件原内容，写入新内容
    public static void writeAll(String path, List<String> data) {
        try (PrintWriter pw = new PrintWriter(new FileWriter(path))) {
            // FileWriter(path) = 打开文件准备写入（默认覆盖模式，会清空原内容）
            // PrintWriter = 打印写入器，提供了 println 方法（写一行并换行）

            // for 循环：遍历列表中的每一行，写入文件
            for (String s : data) {
                pw.println(s);  // 写一行并换行
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }


    // ========== writeLine 方法：追加写入文件 ==========

    // 参数：String path = 文件路径，String line = 要追加的一行内容
    // 效果：在文件末尾添加一行，不清空原有内容
    public static void writeLine(String path, String line) {
        try (PrintWriter pw = new PrintWriter(new FileWriter(path, true))) {
            // FileWriter(path, true) = 第二个参数 true 表示"追加模式"
            // 不加 true 或写 false 就是覆盖模式
            pw.println(line);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### readAllLines 执行过程图解

假设 `user.txt` 内容如下：
```
1,123,qqqqqqqq
2,123456,123456
```

```
readAllLines("user.txt") 被调用
  ↓
打开文件
  ↓
第1次读：line = "1,123,qqqqqqqq"  → 不是空的 → 加入 list
  ↓
第2次读：line = "2,123456,123456" → 不是空的 → 加入 list
  ↓
第3次读：line = null（文件结束）→ 跳出循环
  ↓
返回 list，内容是：
  list[0] = "1,123,qqqqqqqq"
  list[1] = "2,123456,123456"
```

```java
import java.util.List;             // 列表接口
import java.util.stream.Collectors; // Stream 的收集器
import java.util.HashMap;          // HashMap：键值对映射
import java.util.Map;              // Map 接口

public class AddressBookService {

  // loginUserId = 当前登录用户的ID
  // 登录成功后会被赋值，后续所有操作都基于这个ID来过滤数据
  // 初始值是 null，表示还没有用户登录
  private Integer loginUserId = null;


  // ==================== 登录功能 ====================
  // 输入：用户名和密码
  // 返回值：User 对象（登录成功）或 null（登录失败）

  public User login(String username, String pwd) {
    // 第1步：读取 user.txt 的所有内容
    List<String> userData = FileUtil.readAllLines(FileUtil.USER_FILE);

    // 第2步：逐行检查，看有没有匹配的用户名和密码
    for (String line : userData) {
      // line 的格式是 "1,123,qqqqqqqq"
      // split(",") 按逗号拆分成数组：arr[0]="1", arr[1]="123", arr[2]="qqqqqqqq"
      String[] arr = line.split(",");

      // 把字符串形式的ID转成数字
      int id = Integer.parseInt(arr[0]);   // arr[0] 是ID
      String un = arr[1];                   // arr[1] 是用户名
      String pw = arr[2];                   // arr[2] 是密码

      // 如果用户名和密码都匹配
      if (un.equals(username) && pw.equals(pwd)) {
        // 创建一个 User 对象，填充数据
        User u = new User();
        u.setId(id);
        u.setUsername(un);
        u.setPassword(pw);

        // 记录当前登录用户的ID（重要！后续操作都要用到）
        loginUserId = id;

        // 返回用户对象，表示登录成功
        return u;
      }
    }

    // 循环结束都没找到匹配的，返回 null 表示登录失败
    return null;
  }


  // ==================== 注册功能 ====================
  // 输入：用户名和密码
  // 返回值：true（注册成功）或 false（用户名已存在）

  public boolean register(String username, String pwd) {
    // 第1步：读取所有用户数据
    List<String> userData = FileUtil.readAllLines(FileUtil.USER_FILE);

    // 第2步：检查用户名是否已存在
    for (String line : userData) {
      String un = line.split(",")[1];  // 取第2个字段（用户名）
      if (un.equals(username)) {
        return false;  // 用户名重复，注册失败
      }
    }

    // 第3步：生成新的ID
    // ID 的生成策略：当前行数 + 1
    // 比如有3个用户（3行），新用户的ID就是 4
    int newId = userData.size() + 1;

    // 第4步：把新用户信息追加写入文件
    // 格式："4,新用户名,新密码"
    FileUtil.writeLine(FileUtil.USER_FILE, newId + "," + username + "," + pwd);

    return true;  // 注册成功
  }


  // ==================== 查询所有分组 ====================
  // 返回值：所有分组的列表

  public List<Group> getAllGroup() {
    // 创建一个空列表，用来装分组对象
    List<Group> groupList = new ArrayList<>();

    // 读取 group.txt 的所有内容
    List<String> data = FileUtil.readAllLines(FileUtil.GROUP_FILE);

    // 逐行解析，每行变成一个 Group 对象
    for (String line : data) {
      String[] arr = line.split(",");  // 按逗号拆分

      // 创建 Group 对象，设置属性
      Group g = new Group();
      g.setId(Integer.parseInt(arr[0]));          // 第1个字段：ID
      g.setGroupName(arr[1]);                      // 第2个字段：分组名
      g.setIsDefault(Integer.parseInt(arr[2]));    // 第3个字段：是否默认

      // 加入列表
      groupList.add(g);
    }

    return groupList;
  }


  // ==================== 新增分组 ====================
  // 输入：分组名称
  // 返回值：true（成功）或 false（名称重复）

  public boolean addGroup(String name) {
    // 读取所有分组
    List<String> groups = FileUtil.readAllLines(FileUtil.GROUP_FILE);

    // 检查分组名是否重复
    for (String line : groups) {
      String gName = line.split(",")[1];  // 取分组名称字段
      if (gName.equals(name)) {
        // JOptionPane.showMessageDialog = 弹出一个提示对话框
        JOptionPane.showMessageDialog(null, "分组名称重复");
        return false;
      }
    }

    // 生成新ID，追加写入文件
    int newId = groups.size() + 1;
    FileUtil.writeLine(FileUtil.GROUP_FILE, newId + "," + name + ",0");

    JOptionPane.showMessageDialog(null, "分组创建成功");
    return true;
  }


  // ==================== 删除分组 ====================
  // 输入：要删除的分组ID
  // 效果：删除该分组，该分组下的联系人自动移入默认分组

  public void delGroup(int gid) {
    // 第1步：找到默认分组的ID
    List<Group> allGroup = getAllGroup();
    int defaultId = 0;  // 用来存储默认分组的ID

    for (Group g : allGroup) {
      if (g.getIsDefault() == 1) {
        defaultId = g.getId();  // 找到默认分组，记录它的ID
      }
      if (g.getId() == gid && g.getIsDefault() == 1) {
        JOptionPane.showMessageDialog(null, "默认分组禁止删除");
        return;  // 不能删除默认分组
      }
    }

    // 第2步：把属于被删分组的联系人，迁移到默认分组
    List<String> contactData = FileUtil.readAllLines(FileUtil.CONTACT_FILE);
    List<String> newContact = new ArrayList<>();

    for (String line : contactData) {
      String[] arr = line.split(",");
      int gId = Integer.parseInt(arr[9]);  // 第10个字段是分组ID

      if (gId == gid) {
        arr[9] = defaultId + "";  // 把分组ID改成默认分组的ID
      }

      newContact.add(String.join(",", arr));  // 用逗号把数组重新拼成字符串
    }

    // 把修改后的联系人数据写回文件
    FileUtil.writeAll(FileUtil.CONTACT_FILE, newContact);

    // 第3步：从分组文件中删除该分组
    List<String> groupData = FileUtil.readAllLines(FileUtil.GROUP_FILE);

    // 使用 Stream 的 filter 过滤掉要删除的分组
    List<String> newGroup = groupData.stream()
            .filter(s -> Integer.parseInt(s.split(",")[0]) != gid)
            // 这行的意思：只保留 ID 不等于 gid 的行
            // 换句话说：ID 等于 gid 的行被过滤掉了（即被删除了）
            .collect(Collectors.toList());
    // collect = 把过滤后的结果收集回列表

    FileUtil.writeAll(FileUtil.GROUP_FILE, newGroup);

    JOptionPane.showMessageDialog(null, "分组删除完成，联系人移入默认分组");
  }


  // ==================== 新增联系人 ====================
  // 输入：一个 Contact 对象（包含所有字段的值）

  public void addContact(Contact c) {
    // 安全检查：如果用户没有登录，不能新增联系人
    if (loginUserId == null) {
      JOptionPane.showMessageDialog(null, "登录失效，请重新登录！");
      return;
    }

    // 给联系人设置当前登录用户的ID
    c.setUserId(loginUserId);

    // 读取所有联系人，用于生成新ID
    List<String> data = FileUtil.readAllLines(FileUtil.CONTACT_FILE);
    int newId = data.size() + 1;

    // 把 Contact 对象的各字段拼接成逗号分隔的字符串
    // 格式："ID,姓名,电话,省,市,区县,单位,邮箱,备注,分组ID,用户ID"
    String line = newId + "," + c.getName() + "," + c.getPhone() + ","
            + c.getProvince() + "," + c.getCity() + "," + c.getCounty() + ","
            + c.getCompany() + "," + c.getEmail() + "," + c.getRemark() + ","
            + c.getGroupId() + "," + c.getUserId();

    // 追加到文件
    FileUtil.writeLine(FileUtil.CONTACT_FILE, line);

    JOptionPane.showMessageDialog(null, "新增联系人成功");
  }


  // ==================== 查询当前用户的所有联系人 ====================
  // 返回值：Contact 对象的列表

  public List<Contact> queryAllContact() {
    List<Contact> list = new ArrayList<>();

    // 读取所有联系人数据
    List<String> data = FileUtil.readAllLines(FileUtil.CONTACT_FILE);

    // 读取所有分组，建立一个"分组ID → 分组名称"的快速查找表（HashMap）
    List<Group> allGroups = getAllGroup();
    Map<Integer, String> groupNameMap = new HashMap<>();
    for (Group g : allGroups) {
      groupNameMap.put(g.getId(), g.getGroupName());
      // 比如：groupNameMap.put(1, "默认") 表示编号1对应名称"默认"
    }

    // 逐行解析联系人
    for (String line : data) {
      String[] arr = line.split(",");

      // 数据完整性检查：字段不够11个的行是脏数据，跳过
      if (arr.length < 11) {
        continue;  // continue = 跳过本次循环，直接进入下一次
      }

      // 解析用户ID
      int uid;
      try {
        uid = Integer.parseInt(arr[10]);  // 第11个字段是用户ID
      } catch (NumberFormatException e) {
        continue;  // 如果用户ID不是数字（脏数据），跳过
      }

      // 只显示当前登录用户的联系人（用户隔离）
      if (uid != loginUserId) continue;

      // 解析所有字段，创建 Contact 对象
      Contact c = new Contact();
      c.setId(Integer.parseInt(arr[0]));
      c.setName(arr[1]);
      c.setPhone(arr[2]);
      c.setProvince(arr[3]);
      c.setCity(arr[4]);
      c.setCounty(arr[5]);
      c.setCompany(arr[6]);
      c.setEmail(arr[7]);
      c.setRemark(arr[8]);
      c.setGroupId(Integer.parseInt(arr[9]));
      c.setUserId(uid);

      // 通过 HashMap 快速获取分组名称
      // 比如 groupId=3，就从 map 里取 "同学"
      c.setGroupName(groupNameMap.get(c.getGroupId()));

      // 加入结果列表
      list.add(c);
    }

    return list;
  }


  // ==================== 多条件模糊查询 ====================
  // 输入：姓名、电话、省份、单位（四个条件，可以为空）
  // 返回值：匹配的联系人列表

  public List<Contact> searchContact(String name, String phone, String province, String company) {
    // 先获取当前用户的所有联系人
    List<Contact> all = queryAllContact();

    // 使用 Stream 进行过滤
    return all.stream()
            .filter(c -> {
              boolean match = true;  // 默认匹配

              // 如果条件不为空，检查是否包含关键字
              // .contains() 表示"包含"，比如 "张三".contains("张") = true
              if (!name.isEmpty()) match &= c.getName().contains(name);
              if (!phone.isEmpty()) match &= c.getPhone().contains(phone);
              if (!province.isEmpty()) match &= c.getProvince().contains(province);
              if (!company.isEmpty()) match &= c.getCompany().contains(company);
              // &= 表示"并且"：所有填写了的条件都要满足

              return match;  // 返回 true 表示保留这个联系人
            })
            .collect(Collectors.toList());  // 收集回列表
  }


  // ==================== 修改联系人 ====================
  // 输入：一个 Contact 对象（包含新的信息和要修改的ID）

  public void updateContact(Contact target) {
    // 读取所有联系人
    List<String> data = FileUtil.readAllLines(FileUtil.CONTACT_FILE);
    List<String> newData = new ArrayList<>();

    // 逐行检查
    for (String line : data) {
      String[] arr = line.split(",");
      int id = Integer.parseInt(arr[0]);  // 这行的联系人ID

      if (id == target.getId()) {
        // 找到了要修改的联系人！用新数据替换这一行
        String newLine = id + "," + target.getName() + "," + target.getPhone()
                + "," + target.getProvince() + "," + target.getCity()
                + "," + target.getCounty() + "," + target.getCompany()
                + "," + target.getEmail() + "," + target.getRemark()
                + "," + target.getGroupId() + "," + loginUserId;
        newData.add(newLine);
      } else {
        // 不是要修改的联系人，原样保留
        newData.add(line);
      }
    }

    // 把修改后的所有数据写回文件（覆盖）
    FileUtil.writeAll(FileUtil.CONTACT_FILE, newData);

    JOptionPane.showMessageDialog(null, "修改成功");
  }


  // ==================== 删除联系人 ====================
  // 输入：要删除的联系人ID

  public void deleteContact(int cid) {
    // 读取所有联系人
    List<String> data = FileUtil.readAllLines(FileUtil.CONTACT_FILE);

    // 使用 Stream 过滤：只保留 ID 不等于 cid 的行
    // 等于 cid 的那行被过滤掉 = 被删除了
    List<String> newData = data.stream()
            .filter(s -> Integer.parseInt(s.split(",")[0]) != cid)
            .collect(Collectors.toList());

    // 写回文件
    FileUtil.writeAll(FileUtil.CONTACT_FILE, newData);

    JOptionPane.showMessageDialog(null, "删除完成");
  }


  // 获取当前登录用户的ID（供外部使用）
  public Integer getLoginUserId() {
    return loginUserId;
  }
}
```

```java
// 引入图形界面组件

import example.AddressBookService;
import example.MainFrame;
import example.User;

import javax.swing.*;    // Swing 组件（JFrame、JButton、JTextField 等）
import java.awt.*;       // AWT 组件（GridLayout 布局管理器等）

// LoginFrame 继承 JFrame，意思是"LoginFrame 是一种窗口"
// JFrame 提供了窗口的基本能力（标题栏、关闭按钮、大小设置）
// LoginFrame 在此基础上加了登录相关的功能
public class LoginFrame extends JFrame {

  // 创建一个 AddressBookService 对象
  // 这个对象负责处理所有业务逻辑（登录、注册等）
  // 用成员变量而不是局部变量，是因为后续 MainFrame 也要用它
  AddressBookService service = new AddressBookService();

  // 构造方法：创建 LoginFrame 对象时自动执行
  // 方法名和类名一样，没有返回类型
  public LoginFrame() {

    // ---------- 窗口基本设置 ----------

    setTitle("通讯录登录");        // 设置窗口标题
    setSize(330, 200);             // 设置窗口大小：宽330像素，高200像素
    setDefaultCloseOperation(EXIT_ON_CLOSE);
    // 设置关闭窗口时的操作：EXIT_ON_CLOSE 表示关闭窗口时整个程序退出
    setLayout(new GridLayout(4, 2, 10, 10));
    // 设置布局为网格布局：4行2列，水平间距10像素，垂直间距10像素
    // 布局效果：
    // [标签: 账号] [输入框]
    // [标签: 密码] [密码框]
    // [登录按钮]  [注册按钮]
    setLocationRelativeTo(null);
    // 让窗口出现在屏幕正中央（null 表示没有参考对象，默认居中）


    // ---------- 创建界面组件 ----------

    JTextField username = new JTextField();
    // JTextField = 单行文本输入框，用户可以在里面输入文字

    JPasswordField password = new JPasswordField();
    // JPasswordField = 密码输入框，输入的内容会显示为圆点（●●●●●）

    JButton loginBtn = new JButton("登录");
    // JButton = 按钮，括号里的文字显示在按钮上

    JButton regBtn = new JButton("注册新账号");


    // ---------- 把组件添加到窗口 ----------

    // add() 把组件放入窗口
    // 因为布局是 4行2列，每 add 两个组件就自动排一行
    add(new JLabel("账号："));    // 第1行第1列：标签文字
    add(username);                // 第1行第2列：输入框
    add(new JLabel("密码："));    // 第2行第1列：标签文字
    add(password);                // 第2行第2列：密码框
    add(loginBtn);                // 第3行第1列：登录按钮
    add(regBtn);                  // 第3行第2列：注册按钮


    // ---------- 登录按钮点击事件 ----------

    // addActionListener = 给按钮注册一个"点击监听器"
    // 当用户点击按钮时，自动执行 e -> { ... } 里面的代码
    // e 是事件对象（包含点击信息，本项目没用到）
    loginBtn.addActionListener(e -> {

      // 获取用户输入的用户名和密码
      String un = username.getText().trim();
      // .getText() = 获取输入框里的文字
      // .trim() = 去掉首尾空格

      String pw = new String(password.getPassword()).trim();
      // .getPassword() 返回的是 char[]（字符数组），需要转成 String

      // 调用 service 的 login 方法进行登录验证
      User user = service.login(un, pw);

      if (user == null) {
        // 登录失败（返回了 null）
        JOptionPane.showMessageDialog(this, "账号或密码错误！");
        // showMessageDialog = 弹出一个提示框，显示错误信息
      } else {
        // 登录成功
        JOptionPane.showMessageDialog(this, "登录成功");

        this.dispose();
        // dispose() = 销毁当前窗口（关闭登录窗口）

        new MainFrame(service).setVisible(true);
        // 创建主窗口，并显示出来
        // 把 service 对象传给 MainFrame，这样 MainFrame 就能复用同一个 service
        // 复用的原因：loginUserId 保存在 service 对象里，如果创建新的 service，这个ID就丢了
      }
    });


    // ---------- 注册按钮点击事件 ----------

    regBtn.addActionListener(e -> {

      // 创建注册对话框的输入框
      JTextField usernameField = new JTextField();
      JPasswordField passwordField = new JPasswordField();
      JPasswordField confirmPasswordField = new JTextField 是用来输入确认密码的
      // 注意：这里应该是 new JPasswordField()，但原代码写的是 new JTextField()
      // 这是一个小问题：确认密码框不是密码框，输入内容会明文显示


      // 构建对话框的内容（标签 + 输入框的组合）
      Object[] msg = {
              "注册账号：", usernameField,
              "登录密码：", passwordField,
              "确认密码：", confirmPasswordField
      };

      // 弹出一个确认对话框（有"确定"和"取消"两个按钮）
      int result = JOptionPane.showConfirmDialog(
              this,              // 父窗口（当前登录窗口）
              msg,               // 对话框的内容
              "注册新账号",       // 对话框标题
              JOptionPane.OK_CANCEL_OPTION  // 按钮类型：确定 + 取消
      );

      // 用户点击了"确定"
      if (result == JOptionPane.OK_OPTION) {

        // 获取输入的值
        String un = usernameField.getText().trim();
        String pw = new String(passwordField.getPassword()).trim();
        String confirmPw = new String(confirmPasswordField.getPassword()).trim();

        // ---- 校验1：不能为空 ----
        if (un.isBlank() || pw.isBlank() || confirmPw.isBlank()) {
          JOptionPane.showMessageDialog(this, "账号和密码不能为空");
          return;  // return = 直接结束这个方法，不再往下执行
        }

        // ---- 校验2：用户名至少3位 ----
        if (un.length() < 3) {
          JOptionPane.showMessageDialog(this, "账号长度不能少于3位");
          return;
        }

        // ---- 校验3：密码至少6位 ----
        if (pw.length() < 6) {
          JOptionPane.showMessageDialog(this, "密码长度不能少于6位");
          return;
        }

        // ---- 校验4：两次密码一致 ----
        if (!pw.equals(confirmPw)) {
          JOptionPane.showMessageDialog(this, "两次输入的密码不一致");
          return;
        }

        // 所有校验通过，调用 service 注册
        boolean success = service.register(un, pw);

        if (success) {
          JOptionPane.showMessageDialog(this, "注册成功，请登录");
        } else {
          JOptionPane.showMessageDialog(this, "注册失败，该账号已存在");
        }
      }
      // 用户点击"取消"的话，什么都不做（对话框自动关闭）
    });
  }
}
```

```java
    // ---------- 顶部按钮区域 ----------

    JPanel topPanel = new JPanel();

    JButton addBtn = new JButton("新增联系人");

    tableModel = new DefaultTableModel(col, 0);

    JScrollPane scroll = new JScrollPane(table);

    tableModel.setRowCount(0);

  void addDialog() {

    List<Group> groups = service.getAllGroup();
    JComboBox<Group> groupBox = new JComboBox<>();
    // JComboBox = 下拉选择框

    for (Group g : groups) {
      groupBox.addItem(g);  // 每个分组作为一个选项
    }

    // 2. 自定义下拉框的显示格式
    // 默认情况下，JComboBox 会显示对象的 toString() 结果
    // 我们自定义渲染器，让它显示 "分组名(ID)" 的格式
    groupBox.setRenderer(new DefaultListCellRenderer() {
      @Override
      public Component getListCellRendererComponent(
              JList<?> list, Object value, int index,
              boolean isSelected, boolean cellHasFocus) {
        // 调用父类的默认渲染
        super.getListCellRendererComponent(list, value, index, isSelected, cellHasFocus);

        // 如果 value 是 Group 类型，就显示 "分组名(ID)"
        if (value instanceof Group) {
          // instanceof = 判断对象是否是某个类型
          Group g = (Group) value;
          // (Group) = 强制类型转换，把 Object 转成 Group
          setText(g.getGroupName() + "(" + g.getId() + ")");

    });

    // 3. 创建输入框
    JTextField name = new JTextField();
    JTextField phone = new JTextField();
    JTextField pro = new JTextField();      // 省份
    JTextField city = new JTextField();     // 城市
    JTextField county = new JTextField();   // 区县
    JTextField company = new JTextField();  // 单位

  void editDialog() {

    String city = tableModel.getValueAt(row, 4).toString();
    String county = tableModel.getValueAt(row, 5).toString();
    String com = tableModel.getValueAt(row, 6).toString();

    JTextField tfName = new JTextField(name);
    JTextField tfPhone = new JTextField(phone);
    JTextField tfPro = new JTextField(pro);

    JTextField tfCounty = new JTextField(county);

    Object[] msg = {

    int opt = JOptionPane.showConfirmDialog(this, msg, "编辑联系人", JOptionPane.OK_CANCEL_OPTION);

  void deleteContact() {
    // 1. 获取选中的行
    int row = table.getSelectedRow();
    if (row == -1) {
      JOptionPane.showMessageDialog(this, "请选中联系人");
      return;
    }

    // 2. 获取选中行的ID
    Object idValue = tableModel.getValueAt(row, 0);
    if (!(idValue instanceof Integer)) {
      JOptionPane.showMessageDialog(this, "当前没有可删除的联系人");
      return;
    }
    int cid = (Integer) idValue;

    // 3. 二次确认（防止误删）
    int sure = JOptionPane.showConfirmDialog(
            this, "确定删除该联系人？", "确认", JOptionPane.YES_NO_OPTION
    );
    // YES_NO_OPTION = 只有"是"和"否"两个按钮

    if (sure == JOptionPane.YES_OPTION) {
      // 4. 用户确认删除，调用 service 删除
      service.deleteContact(cid);

      // 5. 刷新表格
      refreshTable();
    }
  }


  // ==================== 分组操作 ====================

  void groupOperate() {
    // 1. 弹出输入框，让用户选择操作
    String opt = JOptionPane.showInputDialog("输入1新增分组，输入2删除分组");
    // showInputDialog = 弹出一个带输入框的对话框
    // 如果用户点了"取消"，返回 null

    if (opt == null) return;  // 用户取消了，什么都不做

    if (opt.equals("1")) {
      // 新增分组
      String gname = JOptionPane.showInputDialog("输入分组名称");
      if (gname != null && !gname.isBlank()) {
        service.addGroup(gname);
      }
    } else if (opt.equals("2")) {
      // 删除分组
      String gidStr = JOptionPane.showInputDialog("输入要删除的分组ID");
      try {
        int gid = Integer.parseInt(gidStr);
        // Integer.parseInt = 把字符串转成数字
        // 如果用户输入的不是数字（比如输入了"abc"），会抛出异常
        service.delGroup(gid);
      } catch (Exception e) {
        // 捕获异常：用户输入的不是数字
        JOptionPane.showMessageDialog(this, "ID数字格式错误");
      }
    }
  }


  // ==================== 多条件查询对话框 ====================

  void searchDialog() {
    // 1. 创建查询条件输入框
    JTextField name = new JTextField();
    JTextField phone = new JTextField();
    JTextField pro = new JTextField();
    JTextField com = new JTextField();

    // 2. 组装对话框
    Object[] msg = {
            "姓名", name, "手机号", phone, "省份", pro, "单位", com
    };

    // 3. 弹出查询对话框
    int res = JOptionPane.showConfirmDialog(this, msg, "多条件查询", JOptionPane.OK_CANCEL_OPTION);

    if (res == JOptionPane.OK_OPTION) {
      // 4. 调用 service 查询
      List<Contact> list = service.searchContact(
              name.getText(), phone.getText(), pro.getText(), com.getText()
      );

      if (list.isEmpty()) {
        JOptionPane.showMessageDialog(this, "未找到匹配联系人");
      }

      // 5. 用查询结果刷新表格
      tableModel.setRowCount(0);  // 先清空

      for (Contact c : list) {
        Object[] row = {
                c.getId(), c.getName(), c.getPhone(), c.getProvince(),
                c.getCity(), c.getCounty(), c.getCompany(), c.getEmail(),
                c.getGroupName()
        };
        tableModel.addRow(row);
      }
    }
  }
```

# 第四部分：程序是怎么跑起来的

### 完整运行流程

```
1. 程序启动
   └→ JVM 执行 Main.java 的 main 方法
      └→ new LoginFrame().setVisible(true)
         └→ 登录窗口出现在屏幕上

2. 用户注册（首次使用）
   └→ 点击"注册新账号"
      └→ 弹出注册对话框
         └→ 输入用户名、密码、确认密码
            └→ 点击"确定"
               └→ 校验：非空、长度、密码一致
                  └→ 通过 → service.register() 写入 user.txt → "注册成功"
                  └→ 失败 → 提示对应错误

3. 用户登录
   └→ 输入用户名和密码，点击"登录"
      └→ service.login() 读取 user.txt 比对
         └→ 匹配成功 → 记录 loginUserId → 关闭登录窗口 → 打开主窗口
         └→ 匹配失败 → 提示"账号或密码错误"

4. 进入主界面
   └→ refreshTable() 读取 contact.txt
      └→ 过滤出当前用户的联系人
         └→ 用 HashMap 把分组ID转成分组名称
            └→ 显示在表格中

5. 新增联系人
   └→ 点击"新增联系人"
      └→ 弹出输入对话框
         └→ 填写信息，选择分组
            └→ 点击"确定"
               └→ service.addContact() 追加到 contact.txt
                  └→ refreshTable() 刷新表格

6. 编辑联系人
   └→ 选中表格中的一行
      └→ 点击"编辑联系人"
         └→ 弹出预填当前值的对话框
            └→ 修改内容，点击"确定"
               └→ service.updateContact() 替换 contact.txt 中的对应行
                  └→ refreshTable() 刷新表格

7. 删除联系人
   └→ 选中表格中的一行
      └→ 点击"删除联系人"
         └→ 二次确认对话框
            └→ 点击"是"
               └→ service.deleteContact() 从 contact.txt 移除该行
                  └→ refreshTable() 刷新表格

8. 分组管理
   └→ 点击"分组管理"
      └→ 输入1 → 输入分组名 → service.addGroup() 写入 group.txt
      └→ 输入2 → 输入分组ID → service.delGroup() 删除分组并迁移联系人

9. 多条件查询
   └→ 点击"多条件查询"
      └→ 输入查询条件（可部分填写）
         └→ service.searchContact() 用 Stream filter 过滤
            └→ 用查询结果刷新表格

10. 重置
    └→ 点击"重置"
       └→ refreshTable() 重新显示全部联系人
```

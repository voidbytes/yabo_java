# 第一部分：编程前置知识

在看代码之前，我们先把用到的编程概念全部讲一遍。每个概念会用生活中的例子来类比。

---

## 1.1 什么是编程

**编程**就是给计算机写一份"指令清单"，告诉它一步一步该做什么。就像你写一份菜谱：

```
第一步：打开冰箱，拿出鸡蛋
第二步：把鸡蛋打到碗里
第三步：搅拌均匀
第四步：开火，倒油
第五步：把蛋液倒入锅中
第六步：等30秒，翻面
第七步：关火，装盘
```

计算机就是那个"厨师"，它会严格按照你写的步骤执行。菜谱写错了（比如忘了说开火），做出来的菜就会出问题——程序也是一样。

**Java** 是一种编程语言，就像中文、英文一样，是人和计算机沟通的"语言"。我们用 Java 写的代码，最终会被翻译成计算机能理解的指令。

---

## 1.2 什么是变量

**变量**就是一个"盒子"，用来存放数据。你可以给盒子贴一个标签（变量名），然后把东西放进去。

```
生活中的例子：
你有一个盒子，标签写着"今天的气温"，里面放了一张纸条写着"35度"
```

在 Java 里，写法是：

```java
int temperature = 35;
// int    = 盒子的类型（放整数的盒子）
// temperature = 盒子的标签（变量名）
// 35     = 盒子里放的东西（变量的值）
// ;      = 句号（Java 每句话都要以分号结尾）
```

常用的盒子类型：
| 类型 | 能放什么 | 例子 |
|------|----------|------|
| `int` | 整数 | `int age = 20;` |
| `String` | 一串文字（字符串） | `String name = "张三";` |
| `double` | 小数 | `double price = 9.99;` |
| `Integer` | 整数（对象版） | `Integer id = 1;` |

`int` 和 `Integer` 的区别：`int` 是最基础的数字类型，`Integer` 是它的"升级版"，可以在特殊情况下表示"没有值"（null）。项目里用 `Integer` 是因为数据从文件读出来时可能是空的。

---

## 1.3 什么是数据类型

数据类型就是告诉计算机"这个盒子里要放什么类型的东西"。就像你不能把液体倒进一个方盒子里——容器的形状决定了它能装什么。

Java 有两大类数据类型：

- **基本类型**：`int`、`double`、`char`（单个字符）、`boolean`（真/假）——这些是最简单的值。
- **引用类型**：`String`（文字）、`List`（列表）、你自定义的类（如 `User`）——这些是复杂的数据结构。

---

## 1.4 什么是类和对象

**类（Class）** 是一张"设计图纸"，**对象（Object）** 是根据图纸造出来的"实物"。

```
生活中的例子：
"汽车"是一个类（设计图纸），上面写了汽车有什么特征（品牌、颜色、速度）和能做什么（启动、加速、刹车）。
"我那辆红色的宝马"是一个对象（实物），它有具体的值（品牌=宝马，颜色=红色，速度=0）。
```

在 Java 里：

```java
// 这是"图纸"（类）
public class User {
    String username;   // 特征：用户名
    String password;   // 特征：密码
}

// 这是"实物"（对象）
User u = new User();          // 创建一个空的用户
u.username = "张三";           // 给它的用户名赋值
u.password = "123456";        // 给它的密码赋值
```

- `class` 关键字表示"我要定义一个类"。
- `User` 是类名（图纸的名字）。
- `new User()` 是创建一个对象（按图纸造一个实物）。
- `u.username = "张三"` 是给对象的属性赋值。

---

## 1.5 什么是方法

**方法（Method）** 就是"一段可以反复使用的指令"，类似于菜谱中的一个独立步骤模块。

```
生活中的例子：
"炒鸡蛋"是一个方法：
  输入：鸡蛋、油、盐
  步骤：打蛋→搅拌→热油→倒入→翻炒→出锅
  输出：一盘炒鸡蛋

你可以在不同的菜里调用"炒鸡蛋"这个方法，不用每次都重新写步骤。
```

在 Java 里：

```java
// 定义一个方法
public String greet(String name) {
    return "你好，" + name;
}

// 调用这个方法
String result = greet("张三");
// result 的值变成 "你好，张三"
```

方法的组成部分：
- `public` = 这个方法是公开的，其他地方可以调用它。
- `String` = 这个方法执行完后会返回一个文字结果。
- `greet` = 方法的名字。
- `(String name)` = 输入参数，调用时需要传入的值。
- `return` = 把结果送回去。

## 1.6 什么是 import

`import` 就是"借别人写好的工具"。Java 自带了很多工具库，你需要用 `import` 告诉计算机"我要用这个工具"。

```java
import java.io.BufferedReader;    // 借一个"读文件的工具"
import java.util.ArrayList;       // 借一个"列表工具"
import javax.swing.*;             // 借所有"图形界面工具"（*表示全部）
```

就像做菜时你不会从头造一把菜刀，而是去工具箱里拿一把现成的。

## 1.7 什么是继承

**继承**就是一个类"继承"另一个类的全部特征，然后再加上自己的新特征。

```
生活中的例子：
"动物"有特征：名字、年龄
  ↓ 继承
"狗"有特征：名字、年龄（从动物继承来的）+ 品种（自己新加的）
  ↓ 继承
"导盲犬"有特征：名字、年龄、品种（从狗继承来的）+ 工作证书号（自己新加的）
```

在 Java 里：

```java
public class LoginFrame extends JFrame {
    // LoginFrame 继承了 JFrame 的所有功能
    // JFrame 提供了"窗口"的基本能力（标题栏、关闭按钮、大小设置）
    // LoginFrame 在此基础上加了自己的功能（登录按钮、注册按钮）
}
```

- `extends` 关键字表示"继承自"。
- `JFrame` 是 Java 自带的"窗口类"，提供了创建窗口需要的所有基础功能。
- `LoginFrame` 继承 `JFrame` 后，自动拥有了窗口的能力，只需要关注登录相关的逻辑。

## 1.8 什么是封装

**封装**就是把数据"锁起来"，只留几个"窗口"（方法）来读取和修改。

```
生活中的例子：
银行账户：
  你的余额被锁在银行保险柜里（private，外部不能直接碰）
  你想查余额 → 走"查询窗口"（getter 方法）
  你想存钱   → 走"存款窗口"（setter 方法）
  不能直接去保险柜里改数字
```

在 Java 里：

```java
public class User {
    private String username;  // private = 被锁起来了，外部不能直接访问

    // "查询窗口"：允许外部读取 username 的值
    public String getUsername() {
        return username;
    }

    // "存款窗口"：允许外部修改 username 的值
    public void setUsername(String username) {
        this.username = username;
    }
}
```

- `private` = 私有的，只有这个类内部能访问。
- `public` = 公开的，外部可以调用。
- `getUsername()` = getter 方法，读取值。
- `setUsername(...)` = setter 方法，修改值。
- `this.username` = `this` 指的是"当前这个对象"，`this.username` 就是"当前对象的 username"。

## 1.9 什么是循环

**循环**就是重复做同一件事，直到条件不满足为止。

```
生活中的例子：
"把书架上每本书的书名写到纸上"：
  拿第一本书 → 写书名 → 放回去
  拿第二本书 → 写书名 → 放回去
  拿第三本书 → 写书名 → 放回去
  ...直到没有书了，停止
```

Java 里最常用的循环：

```java
// for 循环：知道要重复几次
for (int i = 0; i < 5; i++) {
    System.out.println("第" + i + "次");
}
// 输出：第0次、第1次、第2次、第3次、第4次

// while 循环：不知道要重复几次，条件满足就一直做
while (还有书) {
    拿一本书 → 写书名 → 放回去
}
```

## 1.10 什么是条件判断

**条件判断**就是"如果……就……，否则……"。

```
生活中的例子：
如果下雨 → 带伞
否则     → 不带伞
```

在 Java 里：

```java
if (下雨) {
    带伞();
} else {
    不带伞();
}

// 也可以用 else if 判断多个条件
if (成绩 >= 90) {
    评级 = "优秀";
} else if (成绩 >= 60) {
    评级 = "及格";
} else {
    评级 = "不及格";
}
```

`==` 表示"等于"，`!=` 表示"不等于"，`>` 表示"大于"，`<` 表示"小于"，`>=` 表示"大于等于"，`&&` 表示"并且"（两个条件都要满足）。

## 1.11 什么是异常处理

程序运行时可能会出错（比如文件不存在、用户输入了文字但代码期望数字）。**异常处理**就是"提前准备好出错时的应对方案"，防止程序直接崩溃。

```
生活中的例子：
你去餐厅点菜：
  正常流程：点菜 → 做菜 → 上菜
  异常情况：点的菜没有 → 告诉你没有这道菜，推荐其他的
  如果没有处理异常：厨房直接起火了（程序崩溃）
```

在 Java 里：

```java
try {
    // 尝试做可能出错的事
    int num = Integer.parseInt("abc");  // 把文字转成数字，"abc"不是数字，会出错
} catch (Exception e) {
    // 出错了怎么办
    System.out.println("输入不是数字！");
}
```

- `try` = "试试看"
- `catch` = "如果出错了"
- `e.printStackTrace()` = 把错误信息打印出来（方便调试）

## 1.12 什么是集合（List、Map）

**集合**就是"可以装很多东西的容器"。

### List（列表）

**List** 就像一个"有序的收纳盒"，东西按顺序排列，可以一个一个拿出来。

```
生活中的例子：
购物清单：
  第1个：牛奶
  第2个：面包
  第3个：鸡蛋
  可以按顺序一个一个看，也可以加新的进去
```

```java
List<String> list = new ArrayList<>();  // 创建一个空列表
list.add("牛奶");                       // 加入第一个
list.add("面包");                       // 加入第二个
list.add("鸡蛋");                       // 加入第三个

String first = list.get(0);  // 取第1个（Java 从0开始数：0、1、2）
// first = "牛奶"

int size = list.size();      // 列表里有几个元素
// size = 3
```

### Map（映射）

**Map** 就像一本"字典"，通过一个词（key）快速找到它的解释（value）。

```
生活中的例子：
字典：
  "苹果" → 一种水果
  "狗"   → 一种动物
  通过"苹果"这个词快速找到"一种水果"
```

```java
Map<Integer, String> map = new HashMap<>();
map.put(1, "默认");   // 放入：编号1 → "默认"
map.put(2, "同学");   // 放入：编号2 → "同学"

String name = map.get(1);  // 通过编号1查找
// name = "默认"
```

## 1.13 什么是 Lambda 表达式

Lambda 表达式就是"一种简写方式"，把一段简短的代码写得更紧凑。

```java
// 传统写法（比较长）
button.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        System.out.println("按钮被点击了");
    }
});

// Lambda 写法（简短）
button.addActionListener(e -> {
    System.out.println("按钮被点击了");
});
```

- `e` 是事件对象（按钮被点击时产生的信息）。
- `->` 表示"执行后面的操作"。
- `{ }` 里面是要执行的代码。

本项目中所有按钮点击事件都用 Lambda 写法。

## 1.14 什么是 Stream API

Stream API 是 Java 8 新增的功能，用来对"一批数据"做批量处理，代码更简洁。

```
生活中的例子：
你有一篮子水果，想找出所有红色的：
  传统方式：一个一个拿出来看，红色的放左边，不是红色的放右边
  Stream 方式：把水果倒进一个流水线，流水线自动把红色的筛选出来
```

```java
// 假设有一个名字列表，要找出所有以"张"开头的
List<String> names = List.of("张三", "李四", "张伟", "王五");

List<String> result = names.stream()       // 把列表变成流水线
    .filter(name -> name.startsWith("张")) // 过滤：只留以"张"开头的
    .collect(Collectors.toList());          // 收集回列表

// result = ["张三", "张伟"]
```

- `.stream()` = 启动流水线。
- `.filter(...)` = 过滤，条件满足的留下。
- `.collect(...)` = 收集结果。
- 本项目在查询和删除操作中大量使用。

## 1.15 什么是图形界面（GUI）

**GUI（Graphical User Interface，图形用户界面）** 就是你在屏幕上看到的窗口、按钮、输入框这些东西。

```
传统程序（命令行）：
  请输入用户名：___  （你在黑色窗口里打字）
  请输入密码：___
  按回车登录

图形界面程序：
  ┌──────────────────┐
  │  账号：[输入框]    │
  │  密码：[输入框]    │
  │  [登录] [注册]     │  ← 用鼠标点击按钮
  └──────────────────┘
```

Java 自带一个叫 **Swing** 的图形界面工具箱，提供了各种现成的组件：

| 组件 | 作用 | 本项目怎么用 |
|------|------|------------|
| `JFrame` | 窗口（整个窗口本身） | LoginFrame、MainFrame 都继承它 |
| `JButton` | 按钮 | "登录"、"新增"等按钮 |
| `JTextField` | 单行输入框 | 输入用户名、姓名等 |
| `JPasswordField` | 密码输入框（输入内容显示为*） | 输入密码 |
| `JLabel` | 标签（显示文字） | "账号："、"密码："等提示文字 |
| `JTable` | 表格 | 显示联系人列表 |
| `JScrollPane` | 滚动条 | 表格内容多时可以滚动 |
| `JComboBox` | 下拉选择框 | 选择分组 |
| `JOptionPane` | 弹出对话框 | 提示信息、确认删除等 |
| `GridLayout` | 网格布局（把组件排成几行几列） | 登录界面的布局 |
| `BorderLayout` | 边界布局（上/下/左/右/中） | 主界面的布局 |

### 数据之间的关系

```
user.id ←── grp.user_id          每个分组属于一个用户（级联删除）
user.id ←── contact.user_id      每个联系人属于一个用户（级联删除）
grp.id  ←── contact.group_id     每个联系人属于一个分组（删除时置 NULL）
```

### 数据库设计思路

#### 为什么拆成三张表而不是一张大表？

如果把所有信息塞进一张表（比如 user_id、group_name、contact_name 全在一起），会产生大量**数据冗余**：

```
反面示例（一张大表）：
张三 | 默认 | user1 | 李四 | 默认 | user1 | 王五 | 同学 | user1
          ↑ "默认"重复了3次，"user1"重复了3次
```

拆成三张表后，每个信息只存一次：

```
user 表：    user1 → 张三
grp 表：     1 → 默认, user1    2 → 同学, user1
contact 表： 1 → 李四, group=1, user=user1
             2 → 王五, group=2, user=user1
```

这就是数据库**第三范式（3NF）**：每张表只描述一个实体，表之间通过外键关联。好处是：修改分组名只需要改一处，不会出现"改了这个名字、漏了那个名字"的不一致问题。

#### 为什么每张表都有 user_id？

通讯录是多用户系统，`grp` 和 `contact` 表都有 `user_id` 字段，实现**用户级数据隔离**：

- 张三的分组（同学、家人）和李四的分组互不影响
- 张三的联系人不会出现在李四的列表里
- SQL 查询时通过 `WHERE user_id = ?` 过滤，数据库层面保证隔离

#### 外键约束的 ON DELETE 策略为什么不同？

| 外键关系 | ON DELETE 策略 | 原因 |
|----------|---------------|------|
| `grp.user_id → user.id` | CASCADE | 删除用户时，该用户的分组也自动删除，不会残留孤立数据 |
| `contact.user_id → user.id` | CASCADE | 删除用户时，该用户的联系人也自动删除，同理 |
| `contact.group_id → grp.id` | SET NULL | 删除分组时，联系人**不删除**，只是分组归属置为 NULL，联系人本身仍有价值 |

如果 `contact.group_id` 也用 CASCADE，删分组时联系人会被误删——这不是我们想要的。

#### VARCHAR(60) 存密码的设计

密码不存明文，而是经过"哈希加盐"处理后存储（详见 3.6 PasswordUtil 章节）。哈希后的结果是一串固定 60 字符的乱码，所以 password 字段用 `VARCHAR(60)` 足够：

```
用户输入密码：123456
  ↓ BCrypt 哈希加盐
存入数据库：  $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhW
              │ │  │ └──────────── 60 字符 ────────────────┘
              │ │  └── 随机盐值（嵌在哈希结果中）
              │ └── 安全等级（10 = 推荐值）
              └── 算法版本

即使数据库被黑客偷走，他看到的也只是这串乱码，无法还原出原始密码。
```

#### username 加 UNIQUE 约束的意义

数据库层面保证用户名唯一，即使业务代码漏了查重检查，数据库也会拒绝插入，不会出现两个同名账号。

程序启动时，`DBUtil` 的静态初始化块会自动执行 `CREATE DATABASE IF NOT EXISTS` 和 `CREATE TABLE IF NOT EXISTS`，无需手动建库建表。

### 为什么要用 private + getter/setter？

用 private 后，只能通过 setter 方法改，你可以在 setter 里加校验逻辑（比如密码长度检查），虽然本项目没加，但这是好的编程习惯。

### groupId 和 userId 是什么意思？

这两个字段是"外键"，用来建立数据之间的关联关系：

```
contact 表的 group_id → 对应 grp 表的 id（分组ID）
contact 表的 user_id  → 对应 user 表的 id（用户ID）

比如：张三的联系人记录
  group_id = 3 → 张三属于第3号分组
  user_id = 1  → 张三属于第1号用户
```

数据库通过外键约束（FOREIGN KEY）自动维护这些关系。删除用户时，该用户的分组和联系人会被级联删除；删除分组时，该分组下的联系人的 `group_id` 会被置为 NULL。

### groupName 为什么不存储在数据库里？

因为 contact 表里存的是 `group_id`（数字3），而界面上需要显示分组名称（"同学"）。查询时通过 `LEFT JOIN` SQL 同时查出联系人和分组名，所以 groupName 是"查询时临时关联出来"的，不需要单独存储。

## 3.5 DBUtil.java — 数据库连接工具

**文件路径**：`src/java/com/example/DBUtil.java`
**总行数**：73 行
**作用**：负责数据库驱动加载、自动建库建表、提供数据库连接。是整个项目的数据层入口。

### 完整代码 + 逐行注释

```java
package com.example;

import java.sql.Connection;     // 数据库连接对象
import java.sql.DriverManager;  // 用来获取数据库连接的工具类
import java.sql.SQLException;   // 数据库操作可能抛出的异常
import java.sql.Statement;      // 用来执行 SQL 语句的对象

public class DBUtil {

    // ========== 连接配置常量 ==========
    private static final String HOST = "jdbc:mysql://192.168.228.100:3306";
    // HOST = MySQL 服务器地址（不带数据库名，用于第一步建库）
    // "jdbc:mysql://" = JDBC 连接 MySQL 的协议前缀
    // "192.168.228.100:3306" = 服务器 IP 和端口号

    private static final String DB = "yabo_address_book";
    // DB = 数据库名称

    private static final String FULL_URL = HOST + "/" + DB
            + "?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8";
    // FULL_URL = 完整的连接地址（带数据库名）
    // useSSL=false = 不使用 SSL 加密（本地开发环境）
    // serverTimezone=Asia/Shanghai = 时区设置为中国上海
    // characterEncoding=utf8 = 字符编码用 UTF-8（支持中文）

    private static final String USER = "root";   // 数据库用户名
    private static final String PWD = "root123";  // 数据库密码


    // ========== 静态初始化块 ==========

    static {
        // 第一步：加载 MySQL 驱动
        // Class.forName() 会触发类的加载，加载时自动注册驱动
        // "com.mysql.cj.jdbc.Driver" 是 MySQL 8 的驱动类名
        // 如果找不到这个类，说明 MySQL 驱动 JAR 没有加入 classpath
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            throw new RuntimeException("MySQL driver not found", e);
            // 抛出运行时异常，程序直接终止（没有驱动什么都做不了）
        }

        // 第二步：自动建库建表
        init();
    }


    // ========== init 方法：自动建库 + 建表 ==========

    private static void init() {

        // --- 第一步：连 MySQL（不带数据库名），创建数据库 ---
        // 因为第一次运行时数据库不存在，不能直接连目标库
        try (Connection conn = DriverManager.getConnection(HOST, USER, PWD);
             // DriverManager.getConnection() = 获取一个数据库连接
             // 传入 HOST（不带数据库名），连的是 MySQL 服务器本身
             Statement st = conn.createStatement()) {
            // Statement = 用来执行 SQL 的对象

            st.executeUpdate("CREATE DATABASE IF NOT EXISTS " + DB
                    + " DEFAULT CHARSET utf8mb4");
            // CREATE DATABASE IF NOT EXISTS = 如果数据库不存在就创建，已存在则跳过
            // DEFAULT CHARSET utf8mb4 = 设置默认字符集为 utf8mb4（支持 emoji 等特殊字符）
        } catch (SQLException e) {
            System.err.println("建库失败: " + e.getMessage());
            return;  // 建库失败，后续建表也无法进行，直接返回
        }

        // --- 第二步：连目标库，创建三张表 ---
        try (Connection conn = DriverManager.getConnection(FULL_URL, USER, PWD);
             // 这次传入 FULL_URL（带数据库名），连接的是 yabo_address_book 数据库
             Statement st = conn.createStatement()) {

            // 创建 user 表
            st.executeUpdate("CREATE TABLE IF NOT EXISTS `user` (" +
                    "id INT AUTO_INCREMENT PRIMARY KEY," +
                    // id = 整数，自动增长，主键（唯一标识每一行）
                    "username VARCHAR(50) NOT NULL UNIQUE," +
                    // username = 最长50字符，不允许为空，不允许重复
                    "password VARCHAR(100) NOT NULL" +
                    // password = 最长100字符，不允许为空
                    ") ENGINE=InnoDB DEFAULT CHARSET=utf8mb4");
            // ENGINE=InnoDB = 使用 InnoDB 存储引擎（支持事务和外键）

            // 创建 grp 表（分组表）
            st.executeUpdate("CREATE TABLE IF NOT EXISTS `grp` (" +
                    "id INT AUTO_INCREMENT PRIMARY KEY," +
                    "group_name VARCHAR(50) NOT NULL," +
                    "is_default TINYINT NOT NULL DEFAULT 0," +
                    // is_default = 小整数，默认值为0
                    "user_id INT NOT NULL," +
                    "FOREIGN KEY (user_id) REFERENCES `user`(id) ON DELETE CASCADE" +
                    // 外键：user_id 关联 user 表的 id
                    // ON DELETE CASCADE = 删除用户时，该用户的分组也自动删除
                    ") ENGINE=InnoDB DEFAULT CHARSET=utf8mb4");

            // 创建 contact 表（联系人表）
            st.executeUpdate("CREATE TABLE IF NOT EXISTS contact (" +
                    "id INT AUTO_INCREMENT PRIMARY KEY," +
                    "name VARCHAR(50) NOT NULL," +
                    "phone VARCHAR(20)," +
                    "province VARCHAR(50)," +
                    "city VARCHAR(50)," +
                    "county VARCHAR(50)," +
                    "company VARCHAR(100)," +
                    "email VARCHAR(100)," +
                    "remark VARCHAR(200)," +
                    "group_id INT," +
                    "user_id INT NOT NULL," +
                    "FOREIGN KEY (group_id) REFERENCES `grp`(id) ON DELETE SET NULL," +
                    // 外键：group_id 关联 grp 表的 id
                    // ON DELETE SET NULL = 删除分组时，联系人的 group_id 置为 NULL
                    "FOREIGN KEY (user_id) REFERENCES `user`(id) ON DELETE CASCADE" +
                    // 外键：user_id 关联 user 表的 id

            System.out.println("数据库初始化完成");  // 控制台打印成功信息
        } catch (SQLException e) {
            System.err.println("建表失败: " + e.getMessage());
        }
    }


    // ========== getConnection 方法：获取数据库连接 ==========

    public static Connection getConnection() throws SQLException {
        // 每次调用都返回一个新的数据库连接
```

## 3.6 PasswordUtil.java — 密码哈希工具

**文件路径**：`src/java/com/example/PasswordUtil.java`
**总行数**：21 行
**作用**：提供密码哈希加盐和验证功能，使用 BCrypt 算法（Spring Security 同款）。

### 为什么不能直接存明文密码？

如果数据库直接存明文密码（比如密码 "123456"，数据库就存 "123456"），一旦数据库被黑客偷走，所有用户的密码全部泄露。更糟糕的是，很多人在不同网站用同一个密码，一个网站泄露可能导致所有账号被盗。

### 什么是哈希加盐？

**哈希** = 把密码变成一串不可逆的乱码。就像碎纸机：文件塞进去变成碎片，碎片无法还原成原来的文件。

**加盐** = 在密码后面随机拼接一串字符，再一起哈希。就像给每道菜加不同的调料：同样的食材（密码），加不同的调料（盐），做出来的菜（哈希值）完全不同。这样即使两个人密码一样，数据库里存的也不一样，黑客没法"查字典"批量破解。

**BCrypt** 是目前最主流的密码哈希算法，Spring Security、Rails、PHP 等框架都在用。它的特点是：自带随机盐（不需要手动管理）、计算速度可调（工作因子越大越慢，让暴力破解成本极高）。

### 完整代码 + 逐行注释

```java
package com.example;

import org.mindrot.jbcrypt.BCrypt;  // jbcrypt 库，BCrypt 的 Java 实现

public class PasswordUtil {

    // BCrypt 工作因子，值越大越安全但越慢，10 是推荐值
    // 每增加1，计算时间翻倍。10 约需 100ms，足够安全
    private static final int WORK_FACTOR = 10;

    // 对明文密码进行哈希加盐
    // BCrypt.hashpw 内部自动生成随机盐，哈希结果包含盐值
    // 返回值形如：$2a$10$N9qo8uLOickgx2ZMRZoMye...（固定60字符）
    public static String hash(String plainPassword) {
        return BCrypt.hashpw(plainPassword, BCrypt.gensalt(WORK_FACTOR));
    }

    // 验证明文密码是否匹配哈希值
    // BCrypt.checkpw 从哈希值中提取盐值，再用相同盐值哈希明文后比较
    public static boolean verify(String plainPassword, String hashedPassword) {
        return BCrypt.checkpw(plainPassword, hashedPassword);
    }
}
```

### BCrypt 工作原理

```
注册时：
  用户输入密码 "123456"
    ↓
  BCrypt.gensalt(10) → 生成随机盐 "$2a$10$KxS0w5..."
    ↓
  BCrypt.hashpw("123456", "$2a$10$KxS0w5...")
    ↓
  哈希结果 "$2a$10$KxS0w5...（盐值）+$9cR5x8...（哈希值）" → 存入数据库

登录时：
  用户输入密码 "123456"
    ↓
  从数据库取出哈希值 "$2a$10$KxS0w5...$9cR5x8..."
    ↓
  BCrypt.checkpw("123456", "$2a$10$KxS0w5...$9cR5x8...")
    ↓
  BCrypt 自动从哈希值中提取盐值，用相同盐值哈希 "123456"，与存储的哈希值比较
    ↓
  匹配 → true（登录成功）/ 不匹配 → false（密码错误）
```

### 为什么选 BCrypt 而不是 MD5/SHA？

| 算法 | 安全性 | 本项目是否使用 |
|------|--------|--------------|
| MD5 | 已被彩虹表破解，不安全 | 否 |
| SHA-256 | 无盐值时仍可被彩虹表攻击 | 否 |
| BCrypt | 自带随机盐，工作因子可调，暴力破解成本极高 | **是** |

```
  ↓
┌─ 校验失败 → 提示对应错误
│
└─ 校验通过 → 调用 service.register()
               ↓
             ┌─ 返回 false → "注册失败，该账号已存在"
             │
             └─ 返回 true  → "注册成功，请登录"
```

```java
  void deleteContact() {
    // 1. 获取选中的行

  void groupOperate() {
    // 1. 弹出输入框，让用户选择操作
    String opt = JOptionPane.showInputDialog("输入1新增分组，输入2删除分组");
    // showInputDialog = 弹出一个带输入框的对话框
    // 如果用户点了"取消"，返回 null

    if (opt == null) return;  // 用户取消了，什么都不做

    if (opt.equals("1")) {
      // 新增分组
      String gname = JOptionPane.showInputDialog("输入分组名称");
      if (gname != null && !gname.isBlank()) {
        service.addGroup(gname);
      }
    } else if (opt.equals("2")) {
      // 删除分组
      String gidStr = JOptionPane.showInputDialog("输入要删除的分组ID");
      try {
        int gid = Integer.parseInt(gidStr);
        // Integer.parseInt = 把字符串转成数字
        // 如果用户输入的不是数字（比如输入了"abc"），会抛出异常
        service.delGroup(gid);
      } catch (Exception e) {
        // 捕获异常：用户输入的不是数字
        JOptionPane.showMessageDialog(this, "ID数字格式错误");
      }
    }
  }

        Object[] row = {
                c.getId(), c.getName(), c.getPhone(), c.getProvince(),
                c.getCity(), c.getCounty(), c.getCompany(), c.getEmail(),
                c.getGroupName()
        };
        tableModel.addRow(row);
      }
    }
  }
```

# 第四部分：程序是怎么跑起来的

### 完整运行流程

```
1. 程序启动
   └→ JVM 执行 Main.java 的 main 方法
      └→ new LoginFrame().setVisible(true)
         └→ 触发 DBUtil 类加载 → 自动建库建表
            └→ 登录窗口出现在屏幕上

2. 用户注册（首次使用）
   └→ 点击"注册新账号"
      └→ 弹出注册对话框
         └→ 输入用户名、密码、确认密码
            └→ 点击"确定"
               └→ 校验：非空、长度、密码一致
                  └→ 通过 → PasswordUtil.hash() 哈希密码 → service.register() 执行 INSERT SQL → 自动创建默认分组 → "注册成功"
                  └→ 失败 → 提示对应错误

3. 用户登录
   └→ 输入用户名和密码，点击"登录"
      └→ service.login() 按用户名查询 → PasswordUtil.verify() 验证密码
         └→ 匹配成功 → 记录 loginUserId → 关闭登录窗口 → 打开主窗口
         └→ 匹配失败 → 提示"账号或密码错误"

4. 进入主界面
   └→ refreshTable() 执行 LEFT JOIN SQL
      └→ 过滤出当前用户的联系人（WHERE user_id = ?）
         └→ 联系人和分组名一次性查出
            └→ 显示在表格中

5. 新增联系人
   └→ 点击"新增联系人"
      └→ 弹出输入对话框
         └→ 填写信息，选择分组
            └→ 点击"确定"
               └→ service.addContact() 执行 INSERT SQL
                  └→ refreshTable() 刷新表格

6. 编辑联系人
   └→ 选中表格中的一行
      └→ 点击"编辑联系人"
         └→ 弹出预填当前值的对话框
            └→ 修改内容，点击"确定"
               └→ service.updateContact() 执行 UPDATE SQL（带 user_id 校验）
                  └→ refreshTable() 刷新表格

7. 删除联系人
   └→ 选中表格中的一行
      └→ 点击"删除联系人"
         └→ 二次确认对话框
            └→ 点击"是"
               └→ service.deleteContact() 执行 DELETE SQL（带 user_id 校验）
                  └→ refreshTable() 刷新表格

8. 分组管理
   └→ 点击"分组管理"
      └→ 输入1 → 输入分组名 → service.addGroup() 执行 INSERT SQL
      └→ 输入2 → 输入分组ID → service.delGroup() 事务：迁移联系人 + 删除分组

9. 多条件查询
   └→ 点击"多条件查询"
      └→ 输入查询条件（可部分填写）
         └→ service.searchContact() 动态拼接 LIKE SQL
            └→ 用查询结果刷新表格

10. 重置
    └→ 点击"重置"
       └→ refreshTable() 重新查询并显示全部联系人
```
